# Security Policy

If you find a security issue, please report it privately to the maintainer instead of opening a public issue.

## Reporting

- Use GitHub security advisories if enabled for the repository
- If that is not available, contact the maintainer directly through GitHub
- Include clear reproduction details and impact

## Expectations

- Do not commit secrets, tokens, API keys, or private data
- Keep only `.env.example` in version control
- Treat local database files and generated artifacts as private by default
